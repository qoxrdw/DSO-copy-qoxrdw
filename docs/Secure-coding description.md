Отчет по проделанной работе:

Реализован MVP, взаимодействие с ним через ключевые эндпоинты. Для взаимодействия можно использовать swagger. Запуск MVP:
	В корне проекта выполнить:
	$env:JWT_SECRET_KEY="your-new-secure-key-from-env"
	uvicorn app.main:app --reload

После запуска перейти в локалхосте на /docs (страница swagger)


### Реализованные контроли безопасности:

### 1) Валидация и нормализация ввода
- Используется Pydantic для автоматической валидации входящих данных по строгой схеме.
- Файлы: shemas.py, main.py
### 2) Ошибки в формате RFC 7807
Реализован механизм маскирования внутренних ошибок (неудачная аутентификация, Rate Limit)
- В файле main.py в обработчике @app.exception_handler(AuthError) используется унифицированное сообщение ошибки UNIFIED_AUTH_ERROR_CONTENT

### 3) Параметризация SQL
Приложение полностью использует SQLAlchemy ORM 2.0 для всех операций с базой данных.
Это значит, что все переменные, передаваемые в запросы (username, title, user_id), автоматически параметризуются
- Файл main.py
### 4) Безопасный HTTP-клиент
Реализован контроль для проверки внешних ссылок (item.link) с использованием асинхронной библиотеки httpx и таймаута (5 секунд).
- файл main.py

### 5) Секреты
Cекретный ключ JWT (SECRET_KEY) был вынесен в переменную окружения (JWT_SECRET_KEY, поэтому его надо указывать до запуска)
- Файл auth_utils.py
