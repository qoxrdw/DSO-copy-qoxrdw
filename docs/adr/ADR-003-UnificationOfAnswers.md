# ADR-001: <RateLimiting>
Дата: 2025-11-03
Статус: Accepted

## Context
NFR-02 (Унификация тела ответа) и NFR-07 (Сокрытие технической информации) требуют, чтобы ответы 401/403 не давали злоумышленнику подсказок.
## Decision
Все неудачные попытки аутентификации (неверный логин, неверный пароль, заблокированный аккаунт) должны возвращать одинаковый HTTP-статус (401) и идентичное тело ответа JSON (например, {"error": "Invalid credentials"}). Ответ не должен содержать детали
## Consequences
Положительные:
1. Предотвращает атаки перечисления пользователей.
2. Соответствует NFR-02 и части NFR-07.

Отрицательные:
1. Затрудняет отладку для разработчиков (придется смотреть логи P1, чтобы понять точную причину ошибки).
2. Необходимо убедиться, что время ответа для разных ошибок также максимально унифицировано, чтобы предотвратить timing attacks.
